## 1. 文档概述

### 1.1 项目背景

"张姐的私房菜谱"是一款面向家庭的微信小程序，旨在解决家庭日常用餐中的"吃什么、谁来做、买什么"三大核心问题。家庭成员可以共同管理菜品库、安排每日菜单、预定想吃的菜、自动汇总采购清单，并通过 AI 获得智能菜品推荐和营养分析。

### 1.2 产品定位

家庭美食管理助手 — 让做饭和吃饭都有条不紊。

### 1.3 目标用户

- **家长（Admin）**：家庭管理者，负责家庭成员管理、菜品库维护、菜单安排
- **大厨（Cook）**：家庭中的烹饪主力，负责菜品添加、菜单执行、采购管理
- **干饭人（Eater）**：一般家庭成员，可预定菜品、查看菜单
- **祖国的花朵（Child）**：儿童成员，权限最低，仅可查看和预定

### 1.4 技术架构

| 层次 | 技术选型 |
|------|---------|
| 前端框架 | 微信小程序原生（WXML + WXSS + JS） |
| 后端服务 | 腾讯云开发 CloudBase（云函数 + 文档数据库 + 云存储） |
| AI 引擎 | DeepSeek API（deepseek-chat / deepseek-v4-pro） |
| 内容安全 | 微信安全 API v2.0（msgSecCheck + imgSecCheck + mediaCheckAsync） |
| 基础库 | ≥ 2.2.3（IDE 3.3.4） |
| 云函数运行时 | Node.js 18.15 |

## 3. 详细功能需求

### 3.1 家庭管理模块

#### F-FAM-01: 创建家庭
- **描述**：未加入家庭的用户可创建新家庭，输入家庭名称后系统自动生成 6 位邀请码
- **前置条件**：用户已登录
- **正常流程**：输入家庭名 → msgSecCheck 检测 → 写入 families 集合 → 用户角色设为 admin
- **异常处理**：名称违规 → Toast 提示修改；安全检查不可用 → 拒绝创建
- **优先级**：P0

#### F-FAM-02: 加入家庭
- **描述**：通过 6 位邀请码加入已有家庭
- **前置条件**：用户已登录且未加入家庭
- **正常流程**：输入邀请码 → 匹配 families 集合 → 用户 family_id 绑定 → 角色设为 eater
- **异常处理**：邀请码无效 → Toast 提示；已加入家庭 → 提示先退出
- **安全设计**：加入固定分配 eater 角色，防止客户端传参提权
- **优先级**：P0

#### F-FAM-03: 成员角色管理
- **描述**：家长（admin）可调整成员角色（admin/cook/eater/child）
- **前置条件**：操作者为 admin，目标成员同家庭
- **正常流程**：选择成员 → ActionSheet 选角色 → family-update 更新
- **安全设计**：最后一位家长不可降级/移除
- **优先级**：P1

#### F-FAM-04: 邀请码分享
- **描述**：复制邀请码到剪贴板，或生成二维码预览
- **优先级**：P2

---

### 3.2 菜品库模块

#### F-DISH-01: 添加菜品
- **描述**：手动填写或 AI 生成菜品信息
- **表单字段**：菜名（必填）、多图（最多9张）、分类、难度、用时、标签、食材清单、做法步骤、是否公开
- **图片安全**：上传→临时云存储→img-check 检测→合规保留/违规删除
- **文本安全**：保存时 msgSecCheck 检测 name+ingredients+steps+tags，fail-close
- **智能安检**：编辑模式下通过表单指纹比对，内容未变跳过重复检测
- **前置条件**：admin 或 cook 权限
- **优先级**：P0

#### F-DISH-02: AI 菜品推荐
- **描述**：根据用餐场景、已有食材、人数、家庭口味/忌口、节气时令、近期历史，AI 推荐 3-5 道菜
- **展示前安全检测**：AI 返回后逐道 msgSecCheck，违规菜品过滤，全部违规则拒绝展示
- **前置条件**：已加入家庭
- **优先级**：P1

#### F-DISH-03: 菜品搜索与筛选
- **描述**：关键词搜索（菜名/食材），分类筛选（家常/硬菜/快手/早餐/汤/甜品/水果/主食/常点/公开）
- **分页**：每页 10 道，触底加载更多
- **游客模式**：无家庭时浏览公开菜品（dish-public），隐藏"常点"分类
- **优先级**：P0

#### F-DISH-04: 批量管理
- **描述**：批量选择 → 修改分类/公开状态/移入回收站
- **前置条件**：admin 或 cook 权限
- **优先级**：P1

#### F-DISH-05: 回收站
- **描述**：软删除的菜品列表，支持恢复和彻底删除
- **前置条件**：admin 或 cook 权限
- **优先级**：P1

#### F-DISH-06: 菜品详情
- **描述**：多图轮播、评分、难度/用时/分类、食材清单、做法步骤
- **三态底部栏**：游客→加入家庭 / 外家菜→引入到我家 / 自家菜→预定+编辑+删除
- **图片预览**：点击图片支持高清大图预览
- **优先级**：P0

#### F-DISH-07: 菜品评分
- **描述**：1-5 星评分，展示平均分，个人评分回显
- **前置条件**：已加入家庭
- **优先级**：P2

### 3.3 菜单管理模块

#### F-MENU-01: 今日菜单
- **描述**：按早餐/午餐/晚餐分组展示今日菜单，显示菜名和做菜人
- **优先级**：P0

#### F-MENU-02: 本周菜单
- **描述**：展示未来 7 天每日安排
- **视角切换**：今日视图 ↔ 本周视图
- **优先级**：P1

#### F-MENU-03: 做菜计时器
- **描述**：点击"开始做菜"→状态变为 cooking → 实时计时器（分:秒）→ 点击"完成"→ done
- **前置条件**：admin 或 cook 权限
- **优先级**：P1

#### F-MENU-04: AI 菜单排期
- **描述**：AI 根据菜品库、本周预定、烹饪历史、成员偏好排出一周每日三餐
- **展示前安全检测**：菜名 msgSecCheck
- **优先级**：P2

### 3.4 预定系统模块

#### F-PRE-01: 预定菜品
- **描述**：选择日期（未来7天）→ 餐次（早/午/晚，默认午餐）→ 多选菜品 → 输入备注 → 提交
- **自动联动**：预定成功自动写入当日菜单 + 当周采购清单
- **前置条件**：已加入家庭
- **优先级**：P0

#### F-PRE-02: 我的预订单
- **描述**：按日期分组展示个人所有预定
- **历史锁定**：今日及未来可编辑/取消，历史预定标记"历史记录"不可操作
- **分页加载**：默认未来+3天历史，底部"加载更早"每次+3天
- **优先级**：P0

#### F-PRE-03: 预定总览
- **描述**：按日期查看全家预定情况，按成员分已预定/未预定
- **管理员操作**：取消他人预定、编辑他人预定
- **提醒功能**：Toast 提示未预定成员
- **优先级**：P1

#### F-PRE-04: 预定编辑
- **描述**：修改已预定的菜品/日期/餐次/备注（先删旧→再建新，失败则回滚）
- **权限**：自己的预定 / 家长管理他人预定
- **优先级**：P1

#### F-PRE-05: AI 营养分析
- **描述**：对指定日期范围的菜单做营养分析（热量、蛋白质/碳水/脂肪比、维生素、均衡评分、建议）
- **展示前安全检测**：summary+suggestions+warnings msgSecCheck
- **优先级**：P2

---

### 3.5 采购管理模块

#### F-SHOP-01: 采购清单查看
- **描述**：按周查看采购清单，按分类分组（蔬菜/肉类/海鲜/调料/主食/水果/蛋类/豆制品/乳制品/其他）
- **统计栏**：总预估花费 + 已勾选/总食材数
- **优先级**：P0

#### F-SHOP-02: AI 采购生成
- **描述**：根据一周菜单食材减去冰箱库存，AI 生成分类采购清单+费用估算
- **展示前安全检测**：物品名+总结 msgSecCheck
- **前置条件**：admin 或 cook 权限
- **优先级**：P1

#### F-SHOP-03: 勾选购买
- **描述**：点击食材项切换已买/未买状态，实时同步云端
- **优先级**：P0

#### F-SHOP-04: 手动管理
- **描述**：添加食材（名称+用量+分类）、删除食材、全选/取消全选
- **前置条件**：admin 或 cook 权限（写入操作）
- **优先级**：P1

#### F-SHOP-05: 批量编辑
- **描述**：进入批量模式 → 多选食材 → 批量删除
- **前置条件**：admin 或 cook 权限
- **优先级**：P2

#### F-SHOP-06: 清单分享
- **描述**：格式化为文本（分类+食材名+用量+勾选状态+预估花费）复制到剪贴板
- **优先级**：P2

### 3.6 个人中心模块

#### F-PRO-01: 宾客登录
- **描述**：未登录用户使用微信 chooseAvatar + type="nickname" 设置头像和昵称
- **图片安全**：头像上传后 login 云函数 imgSecCheck 检测，违规拒绝
- **乐观更新回退**：安全检测失败时回退本地头像/昵称到旧值
- **优先级**：P0

#### F-PRO-02: 个人资料编辑
- **描述**：修改昵称（弹窗输入）、更换头像（选图→压缩→上传→imgSecCheck）
- **乐观更新回退**：检测失败回退本地状态
- **优先级**：P1

#### F-PRO-03: 口味偏好
- **描述**：4 个滑块设置辣/甜/酸/咸度（0-5），1 秒防抖保存
- **优先级**：P2

#### F-PRO-04: 忌口管理
- **描述**：输入添加忌口标签，点击标签移除
- **文本安全**：每项忌口 msgSecCheck 检测
- **乐观更新回退**：检测失败从列表移除
- **优先级**：P2

#### F-PRO-05: 个人统计
- **描述**：展示菜品数、做菜次数、预定次数
- **优先级**：P2

#### F-PRO-06: 意见反馈
- **描述**：弹窗输入反馈内容（2-500字），提交到 feedback 集合
- **文本安全**：反馈内容 msgSecCheck 检测
- **前置条件**：已登录
- **优先级**：P2

#### F-PRO-07: 退出登录
- **描述**：确认后清除所有本地缓存和全局状态，reLaunch 到首页
- **优先级**：P1

#### F-PRO-08: 关于
- **描述**：弹窗显示版本号 + ICP 备案号
- **ICP 备案号**：沪ICP备2026029453号-1X
- **优先级**：P2

## 6. 内容安全策略

### 6.1 安全检测覆盖

| 检测类型 | API | 版本 | 涉及云函数 | 策略 | 违规提示方式 |
|---------|-----|------|-----------|------|------------|
| 文本安全 | `security.msgSecCheck` | v2.0 | 12 个 | fail-closed | Modal 弹窗 |
| 图片安全 | `security.imgSecCheck` | v2.0 | 4 个 | 正常返回 errCode≠0 或抛异常 errCode=87014 | Modal 弹窗 |
| 异步媒体安全 | `security.mediaCheckAsync` | v2.0 | 1 个 | fail-close | — |

### 6.2 fail-closed 模式

所有 msgSecCheck 调用统一使用 fail-closed 判断模式：
```
passed = check.result && check.result.suggest === 'pass'
if (!passed) → 拒绝
```
当 API 返回缺少 `result` 字段时（权限过渡期可能出现），`passed` 为 `false`，内容被拒绝。

### 6.3 imgSecCheck 87014 处理

`imgSecCheck` 和 `msgSecCheck` 行为不同：检测到违规时 `imgSecCheck` 以抛异常（errCode=87014）方式返回，而非正常返回 errCode。catch 块需特殊识别 87014：

```
catch (e) {
    if (e.errCode === 87014) → 返回"头像包含违规内容"
    其他 → 返回"安全检查暂时不可用"
}
```

### 6.2 检测场景

| 场景 | 检测时机 | 检测内容 |
|------|---------|---------|
| 昵称 | 登录/编辑时 | 用户昵称文字 |
| 头像 | 登录/更换时 | 头像图片内容 |
| 菜品 | 添加/编辑保存时 | 菜名、食材名、步骤、标签 |
| 菜品图片 | 上传时 | 每张图片逐张检测 |
| 家庭名 | 创建/编辑时 | 家庭名称 |
| 忌口 | 添加时 | 忌口食材名 |
| 预定备注 | 提交/修改时 | 备注文字 |
| 采购物品 | 创建/更新时 | 物品名称 |
| 菜单备注/图片 | 评分时 | 备注文字+图片 |
| AI 生成内容 | 返回前端展示前 | 菜名/步骤/贴士/营养分析/排期/采购清单 |
| 反馈 | 提交时 | 反馈文字 |

### 6.3 乐观更新回退

前端在以下场景实施了乐观更新+安全检测失败回退机制：
- 登录头像/昵称 → 检测失败回退为空/默认值
- 换头像 → 检测失败回退为旧头像
- 改昵称 → 检测失败回退为旧昵称
- 添加忌口 → 检测失败从列表移除

## 8. 业务流程

### 8.1 核心流程：从预定到采购

```
成员预定菜品 → 自动写入当日菜单 + 当周采购清单
    → 家长/大厨查看采购清单
    → AI 根据菜单汇总食材，减去冰箱库存
    → 生成最终采购清单 → 成员逐项购买勾选
```

### 8.2 核心流程：AI 辅助决策

```
用户输入场景+食材 → AI 推荐菜品
    → 展示前 msgSecCheck 安全过滤
    → 用户选择入库 → dish-add msgSecCheck 复查
    → 菜品加入菜品库
```

### 8.3 用户生命周期

```
首次打开 → 宾客模式（浏览公开菜品）
    → 微信登录（设置头像昵称）
    → 创建/加入家庭
    → 开始使用完整功能
    → 退出登录 → 回到宾客模式
```
